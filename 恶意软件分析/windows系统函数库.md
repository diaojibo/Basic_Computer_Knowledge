## windows系统函数库
windows系统支援一种被称为动态连结库的共享程式库。它只需要被载入内存一次，即可被多于一个处理程序使用。

### 内部元件

#### Hal.dll
Windows系统的硬件抽象层就是由Hal.dll实现。HAL提供很多函数，而这些函数在不同的硬件平台（以下皆指芯片组）皆有不一样的实现方式。因为Windows提供HAL这一个功能，所以大部分程式可以随意呼叫这些函数，而不需要顾及程序在何种平台上执行。

#### Ntdll.dll
Ntdll.dll和ntoskrnl.exe里含有Windows的原生API(native api)，通常被一些必须要在Win32子系统以外的环境下执行的应用程序使用，而这些程式也被称为原生应用程序。大部分API函式的名字通常以Nt开头，例如NtDisplayString。Ntdll.dll除了被原生应用程序呼叫外，它还会被Kernel32.dll里大部分API所使用。很大部分的视窗应用程序也不会直接呼叫Ntdll.dll。

可以理解为windosw内核提供的所有原生API都是从这个DLL中导出的。。。例如你写一个exe程序，需要调用kernel32.dll中的ReadFile()函数，但是实际上最终kernel32.dll中的ReadFile()又调用了ntdll.dll中的NtReadFile()函数。。。所以说ntdll是一扇门，通过调用它你可以让你的代码运行到内核当中。

### Win32 API

#### Kernel32.dll
Kernel32.dll提供应用程序一些Win32下的底层API，包括内存管理、输入/输出操作和同步函数。

#### Gdi32.dll
Gdi32.dll提供跟图形装置界面有关的函式，例如输出到显卡和打印机的原生绘图功能。呼叫这个程式库里函式的应用程序通常是为了执行底层的绘图功能、文字输出、字型管理或其他相似功能。

#### User32.dll
User32.dll提供创建和管理Windows图形界面的功能，例如桌面、视窗和功能表。里面的函式可以让应用程序建立及管理视窗、接收Windows讯息（诸如使用者的输入或系统的通知）、在视窗中显示文字，及显示一个讯息视窗。
这个函式库里面大部分函式也需要倚靠Gdi32.dll提供的绘图功能，来对使用者界面进行渲染。有些程式还会直接使用GDI函式，来对先前由User32.dll所建立的视窗进行底层绘图。
