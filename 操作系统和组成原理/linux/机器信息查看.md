## linux机器信息查看

### 查看cpu信息

可以用lscpu命令

```
blue@blue-pc:~$ lscpu
Architecture:          i686            #cpu架构
CPU op-mode(s):        32-bit, 64-bit
Byte Order:            Little Endian   #小尾序
CPU(s):                4               #总共有4核
On-line CPU(s) list:   0-3
Thread(s) per core:    1               #每个cpu核，只能支持一个线程，即不支持超线程
Core(s) per socket:    4               #每个cpu，有4个核
Socket(s):             1               #总共有1一个cpu
Vendor ID:             GenuineIntel    #cpu产商 intel
CPU family:            6
Model:                 42
Stepping:              7
CPU MHz:               1600.000
BogoMIPS:              5986.12
Virtualization:        VT-x            #支持cpu虚拟化技术
L1d cache:             32K
L1i cache:             32K
L2 cache:              256K
L3 cache:              6144K
```

了解每个cpu型号：

可用cat /proc/cpuinfo

```
#cat /proc/cpuinfo
processor    : 0
vendor_id    : GenuineIntel
cpu family    : 6
model        : 42
model name    : Intel(R) Core(TM) i5-2320 CPU @ 3.00GHz
.....
```

### 内存

```
free -m
             total       used       free     shared    buffers     cached
Mem:          3926       3651        274          0         12        404
-/+ buffers/cache:       3235        691
Swap:         9536         31       9505
```

这里的单位是MB，总共的内存是3926MB。

查看内存详细的使用情况：

```
# cat /proc/meminfo 
MemTotal:        4020868 kB
MemFree:          230884 kB
Buffers:            7600 kB
Cached:           454772 kB
SwapCached:          836 kB
.....
```

### 磁盘

```
# lsblk
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINT
sda      8:0    0 465.8G  0 disk 
├─sda1   8:1    0     1G  0 part /boot
├─sda2   8:2    0   9.3G  0 part [SWAP]
├─sda3   8:3    0  74.5G  0 part /
├─sda4   8:4    0     1K  0 part 
├─sda5   8:5    0 111.8G  0 part /home
└─sda6   8:6    0 269.2G  0 part 
```

用lsblk显示得很直观。上面只给出了硬盘分区的空间大小，使用情况我们还不知道，我们需要用`df` 命令来查看硬盘空间使用情况。

```
-bash-4.1$ df -h
Filesystem            Size  Used Avail Use% Mounted on
/dev/sda1             9.9G  3.2G  6.2G  34% /
/dev/sda3              20G  9.3G  9.5G  50% /usr/local
/dev/sda4             244G  155G   77G  67% /data
tmpfs                  32G     0   32G   0% /dev/shm
```

上图我们可知每个sda大小多少，used多少，挂载在哪个目录。
